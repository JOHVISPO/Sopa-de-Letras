Definiciones

  • Software: Conjunto de programas, instrucciones y reglas informáticas para ejecutar tareas en una computadora.
  • Robustez: Un SW es robusto si se comporta razonablemente aún bajo circunstancias no previstas.
  • Eficiencia: Un SW es eficiente si utiliza los recursos en forma económica.
  • Amigabilidad: Un SW es amigable si es fácil de usar para el usuario final.
  • Verificabilidad: Un SW es verificable si fácilmente se puede ver si se cumplen sus propiedades o cualidades.
  • Mantenibilidad: Un SW es mantenible si es factible realizarle modificaciones fácilmente una vez liberado.
  • Reusabilidad: Un SW es reusable si puede utilizarse con cambios mínimos para hacer un SW nuevo.
  • Portabilidad: Un SW es portable si puede ejecutarse en diferentes plataformas o ambientes.
  • Comprensibilidad: Un SW es comprensible si es fácil de ser entendido, por los desarrolladores y por los usuarios finales.
  • Interoperabilidad: Un SW es ínter operable si es capaz de coexistir y cooperar con otros productos de software.
  • Productividad: El proceso de producción de SW es un proceso productivo cuando mide su propia eficiencia.
  • Oportunidad: El proceso de desarrollo de SW es oportuno cuando es capaz de entregar el producto a tiempo.
  • Visibilidad: El proceso de desarrollo de SW y el propio SW es visible si todos sus pasos y su estado actual están claramente documentados.
